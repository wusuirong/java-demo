factory
父类工厂定义抽象的工厂方法，子类工厂去负责实现工厂方法
这样客户使用时通过更换不同的子类工厂就能生产相应的产品

屏蔽new对象的方式，让客户能针对接口编程
这个模式也说明，设计模式不是创造一个能以不变应万变的模式，而是把变化封装起来，让以后容易扩展维护。

工厂方法与简单工厂的区别
你编写一个简单工厂后，编译出class提供给客户使用。当客户需要新的产品时，你得修改简单工厂源码，重新编译给客户
工厂方法的父类定义了生产产品的框架，实现由子类去做。当客户需要新的产品时，可以不修改源码，增加工厂子类和产品子类即可，这就是对修改关闭，对扩展开放

抽象工厂就是能生产一组相关产品的工厂方法模式
